## 1. 产品概述

### 1.1 产品定位

SmartStock 是一款面向 A 股个人投资者的**盘中实时交易辅助工具**，将《顶级股票交易系统》的规则化交易体系转化为可交互的 Web 应用。系统不提供自动交易，而是作为「纪律执行器」，帮助用户在盘中完成：判断大盘环境 → 策略选股 → 仓位计算 → 交易记录 → 复盘分析 的完整闭环。

### 1.2 目标用户

- 有一定交易经验但缺乏纪律性的 A 股个人投资者
- 希望将主观交易转化为系统化交易的散户
- 已阅读或正在学习结构化交易规则的进阶投资者

### 1.3 核心价值

| 维度 | 价值 |
|------|------|
| 纪律 | 强制执行交易前检查清单、止损规则、仓位上限 |
| 效率 | 一站式完成从大盘判断到下单决策的全流程 |
| 复盘 | 自动统计胜率/盈亏比，识别违规模式和连续亏损 |
| 学习 | 策略速查页面随时回顾规则，降低决策犹豫 |

### 1.4 技术架构

| 层 | 技术 | 说明 |
|----|------|------|
| 前端 | Vue 3.5 + Vite 6 + Vue Router 4 + Pinia 2 | SPA，Composition API |
| 后端 | Koa 2 + @koa/router + @koa/cors | 纯 API 代理层，无数据库 |
| 数据源 | 东方财富公开 API + macroview.club + 证券时报/腾讯/新浪备用 | 通过后端代理避免 CORS，主用东方财富，失败自动切换；社融数据来自 macroview.club |
| 持久化 | localStorage | 用户数据全部本地存储 |
| 部署 | 静态部署 | `vite build` + Node 起后端即可 |

## 3. 功能模块详细需求

### 3.1 大盘状态 (Dashboard)

**路由**: `/`
**用途**: 盘前/盘中判断当前市场环境，决定是否开仓及仓位上限

#### 3.1.1 九维判据表（v7.3）

系统自动采集 9 个维度数据，采用**加权评分**机制逐条判断牛/震/熊信号。v7.3 改进：新增波动率维度（ATR归一化+趋势）、涨跌停集中度惩罚方向修正、阈值悬崖平滑化（线性插值）、冷却期改用交易日计数。

| 维度 | 权重 | 数据来源 | 牛市信号 | 熊市信号 |
|------|------|----------|----------|----------|
| MACD | 1.0 | 三指数(上证/深证/创业板) MACD(12,26,9) 多数投票 | 三指数多数看多（三者同向 ×1.2，两同向 ×1.0，分化 ×0.7） | 三指数多数看空 |
| 涨跌家数 | 1.5 | 沪深京A股实时涨跌统计 | 上涨/下跌 ≥ 2（强牛 2.0×），≥ 1.8 偏强（1.5×），≥ 1.5 略偏多（1.05×） | 下跌/上涨 ≤ 0.5（强熊 2.0×），≤ 0.55 偏弱（1.5×），≤ 0.67 略偏空（1.05×） |
| RSI | 1.0 | 三指数 RSI(14) 多数投票 | 三指数多数看多 | 三指数多数看空 |
| 融资余额 | 1.2 | 近 10 日融资余额线性回归 | 回归斜率 > +0.3%/日 | 回归斜率 < -0.3%/日 |
| 量价配合 | 1.3 | 三指数 OBV 趋势 + 背离 + 成交额/量显示 | 三指数多数看多 | 三指数多数看空 |
| 北向资金 | 1.5 | 近 20 日北向资金成交额 | 5 日均量 / 20 日均量 ≥ 1.2 | 5 日均量 / 20 日均量 ≤ 0.8 |
| 涨跌停 | 1.3 | 当日涨跌停统计 + 连板 + 板块分布 | 并行评分 net ≥ 2 | 并行评分 net ≤ -2 |
| 宏观因子 | \|score\| × 0.5 | PMI/M1-M2剪刀差/CPI/GDP/社融 | 宏观评分 > 0 | 宏观评分 < 0 |

**涨跌停维度（并行评分机制 v7.2）**:

涨跌停维度采用独立的并行评分系统，同时计算牛信号分（bullPts）和熊信号分（bearPts），避免串行 if-else 导致的"牛信号掩盖熊信号"bug。v7.2 新增 4 项子评分和 2 个惩罚系数：

| # | 评分因子 | 牛信号条件 | 熊信号条件 |
|---|---------|-----------|-----------|
| 1 | 涨跌比 | ≥5:1 (+3), ≥2.5:1 (+2), ≥1.5 (+1) | ≤0.2 (-3), ≤0.4 (-2), <0.8 (-1) |
| 2 | 跌停绝对数 | — | ≥100 (+3), ≥50 (+2), ≥20 (+2), ≥10 (+1) |
| 3 | 涨停+封板质量 | ≥80 且封板≥70% (+3), ≥80 且≥40% (+2), ≥40 且≥50% (+2), ≥40 且≥40% (+1), ≥30 且≥50% (+1) | — |
| 4 | 打板次日收益 | ≥+2% (+1) | ≤-2% (+2), <-0.5% (+1) |
| 5 | 低封板率 | — | 封板率<40% 且涨停≥5 (+1) |
| 6 | 低赚钱效应 | — | 赚钱效应<30% (+1) |
| 7 | 高赚钱效应 | 赚钱效应≥60% (+1) | — |
| 8 | **连板统计** | 连板股≥10 (+2), ≥5 (+1), 最高连板≥5天 (+1) | — |
| 9 | **板块集中度惩罚** | — | 涨停集中在同一板块占比≥30% → 看多信号 ×0.6 |
| 10 | **自然涨停率惩罚** | — | 自然涨停率<50%（含ST/新股）→ 看多信号 ×0.8 |
| 11 | **涨停趋势** | 当日涨停 ≥ 3日均值×2 (+1) | 当日涨停 ≤ 3日均值×0.3 (+1) |

**涨跌比特殊处理**: 当跌停为 0 时，不直接使用涨停数作为比值，而是按涨停量级映射：≥30 映射为 5，≥10 映射为 2.5，≥3 映射为 1.5，其他映射为 1，避免"5 家涨停 0 跌停"被判为"极强"的失真。

最终 net = bullPts − bearPts，再经过两个惩罚系数修正：板块集中度过高（≥30%）时看多信号 ×0.6，自然涨停率过低（<50%）时看多信号 ×0.8。修正后：net ≥ 4 为强牛（权重 ×1.5），net ≥ 2 为牛，net ≤ -4 为强熊（权重 ×1.5），net ≤ -2 为熊。

**MACD/RSI/量价维度改进（v7.2 多指数覆盖）**:
- **多指数综合判定**: MACD、RSI、量价三个技术维度对上证/深证/创业板分别独立计算，通过投票机制综合输出
  - 三指数同向 → 权重 ×1.2（强确认）
  - 两同向一反向 → 权重 ×1.0（多数意见）
  - 三者分化 → neutral，权重 ×0.7（弱化）
  - 信号卡片展示各指数方向标签
- **增强背离检测**: 多尺度极值检测（2/3/5-bar 窗口合并去重）、幅度过滤（价格变化需 ≥ 日均波动率 ×0.5）、时间衰减置信度（半衰期 14 bar）
  - 背离信号返回置信度 0.1~1.0，权重 = W_STRONG × confidence
  - 信号描述显示置信度百分比（如"底背离(85%)"）
- **真实交叉检测**: 通过柱状图（hist）穿越零轴判断金叉/死叉，取代旧的 DIF>DEA 静态位置判断
- **动量衰减 ≠ 方向反转**: DIF>DEA 且 DIF>0 但柱状图收缩（多头减弱）判为 neutral 而非 bear；对称地，空头减弱也判为 neutral
- **弱信号降权**: 零轴下方的金叉和零轴上方的死叉，权重降为 W.macd × 0.7
- **零轴震荡**: DIF 和 DEA 方向与零轴不一致时返回 neutral

**RSI 维度改进**:
- **高位横盘仍偏多**: RSI≥60 即使 5 日趋势不向上，仍判为 bull（偏强，权重 ×0.7）
- **超卖回升门槛提高**: 从 RSI>40 提高到 RSI>45，避免 RSI 41（仍偏弱）被判为多头信号
- **空头梯度**: RSI<35 为"弱势"（W.rsi），RSI<40 为"偏弱"（W.rsi × 0.7），区分严重程度

**量价配合维度改进**:
- **归一化斜率 + 死区**: OBV 和价格斜率归一化为日均变化率，设 ±0.05% 死区（≈20 天累计 1%），过滤横盘噪声
- **中性出口**: 斜率不显著时返回 neutral"量价平稳"，不再强制归入 bull/bear
- **标签修正**: "缩量吸筹"改为"低位吸筹"（OBV 上升≠缩量）
- 顶背离：价格创新高但指标未创新高 → 强熊信号（权重 ×1.5）
- 底背离：价格创新低但指标未创新低 → 强牛信号（权重 ×1.5）

**宏观因子维度（第 8 维）**:
- 服务端从东方财富数据中心获取 CPI/PMI/M2/GDP 数据，从 macroview.club 获取社融数据
- 5 个子因子独立评分后累加，clamp 到 [-2, +2]
- 作为修正项加入九维判据：`weight = abs(macroScore) * 0.5`

| 因子 | 正面条件 | 负面条件 | 分值 |
|------|----------|----------|------|
| PMI | > 50.5（扩张区间） | < 49.5（收缩区间） | ±1 |
| M1-M2 剪刀差 | 较上月收窄（企业活力增强） | 较上月扩大（企业活力下降） | ±1 |
| CPI | 0-2%（温和） | > 2%（通胀压力 -0.5），< 0%（通缩 -1） | -1~-0.5 |
| GDP | > 5.5%（偏强） | < 4.5%（偏弱） | ±0.5 |
| 社融 | 年增量同比 > 5%（信用扩张） | < -5%（信用收缩） | ±0.5 |

**后端接口**:
- `GET /api/market/indices` — 上证/深证/创业板指数 + 120 日 K 线 + MA20/60/120
- `GET /api/market/breadth` — 沪深京A股涨/跌/平家数（东方财富 ulist API，单请求汇总沪A+深A+北交所）
- `GET /api/market/northbound` — 近 20 日北向资金成交额（datacenter-web RPT_MUTUAL_DEALAMT）
- `GET /api/market/margin` — 近 10 日融资余额数据
- `GET /api/market/limit-stats` — 当日涨跌停统计（涨停数、跌停数、自然涨停、曾涨停、封板率、赚钱效应、打板次日收益、连板统计、涨停板块分布）
- `GET /api/analysis/macro` — 宏观因子评分（PMI/M1-M2剪刀差/CPI/GDP/社融），评分范围 [-2, +2]
- `GET /api/analysis/sectors` — 板块排名 + RS轮动信号 + TOP5强势/弱势板块
- `GET /api/analysis/valuation` — 估值分析（PE/MA250/股债利差/52周高低）

#### 3.1.2 综合判定

采用**加权评分**机制，各维度乘以对应权重后分别累加至牛信号总权重（bullW）和熊信号总权重（bearW），再结合**状态惯性（hysteresis）**机制判定市场状态：

**权重配置**:
```
W = { macd: 1.0, breadth: 1.5, rsi: 1.0, margin: 1.2, volumePrice: 1.3, northbound: 1.5, limitStats: 1.3 }
W_STRONG = 2.0  // 强信号（背离、极端比值等）额外 ×2.0，背离信号乘以置信度 0.1~1.0
MACRO = abs(macroScore) * 0.5  // 宏观因子修正项，由服务端计算
```

**阈值判定**（基于 8 维度总权重校准）:

| 状态 | 条件 | 建议总仓位 | 推荐策略 |
|------|------|------------|----------|
| 牛市 | bullW ≥ 4.5 且 net > 0 | 80-100% | 趋势突破 |
| 偏多 | bullW ≥ 3.0 且 net > 0 | 50-70% | 回调买入 |
| 震荡 | 其他 | ≤50% | 回调买入 |
| 偏空 | bearW ≥ 3.0 且 net < 0 | 10-20% | 仅观望 |
| 熊市 | bearW ≥ 4.5 且 net < 0 | 0-20% | 空仓 |

其中 net = bullW − bearW。

**状态惯性（hysteresis v7.2）**:
- 同方向调整（牛市↔偏多、熊市↔偏空）自由切换
- 跨方向切换（多头↔空头）：需 |net| ≥ 1.5 + 连续 2 个交易日确认（crossCount），否则保持上一状态
- 跨方向翻转冷却期：翻转后 3 个日历日内禁止反向翻转（lastFlipDate）
- 进出中性：需 |net| ≥ 1.5，否则保持上一状态
- 前一状态（含 crossCount、lastFlipDate）持久化到 localStorage，页面刷新后恢复

**确认机制**: 当领先方向的加权分 ≥ 3.5 且领先于对方时 `confirmed = true`（即 `(bullW ≥ 3.5 && bullW > bearW) || (bearW ≥ 3.5 && bearW > bullW)`）。

**数据来源**:
- 前一市场状态: localStorage `market_prev_status`
- 涨跌停历史: market store `limitStats_cache`

#### 3.1.3 策略选股建议

根据九维判据结果自动匹配策略，**复用选股筛选页面的四层漏斗逻辑**生成自然语言选股条件（keyWordNew 字符串），优先通过东财 AI 自然语言选股 API 实时查询候选股票，AI 解析不完整时自动回退到结构化选股 API。选股条件中加入 RS 强势行业过滤（取板块轮动 TOP3 强势行业）。

**四层漏斗预设**（由 `getStrategyPreset()` 根据市场状态自动填充，支持严/宽两种模式）：

| 市场状态 | 第一层排雷 | 第二层基本面（严/宽） | 第三层景气度 | 第四层技术信号 |
|---------|-----------|------------|------------|-------------|
| 牛市 | 8 项全选（auto 项） | ROE≥12/8, 营收增速≥10/5, 利润增速≥10/5, 负债率≤60/65, PE 5~40 / 3~60, 市值≥50/30亿 | 机构增持 | 趋势突破（MACD金叉+均线多头排列） |
| 偏多 | 8 项全选 | ROE≥12/8, 营收增速≥0/5, 利润增速≥0/5, 负债率≤60/65, PE 5~40 / 3~60, 市值≥50/30亿 | 无 | 回调买入（均线多头排列+量能收缩） |
| 震荡 | 8 项全选 | ROE≥12/8, 营收增速≥0/5, 利润增速≥0/5, 负债率≤60/65, 现金流+(严)/无(宽), PE 5~30 / 3~50, 市值≥50/30亿 | 业绩超预期 | 回调买入（严：均线多头+明显缩量）/ 底部确认（可切换） |
| 偏空/熊市 | — | — | — | 不生成选股建议 |

排雷层固定选中所有 `auto: true` 的 8 项（非ST、非停牌、非北交所、非退市、审计意见、商誉<30%、质押<60%、上市>1年），与选股筛选页面一致。

**AI 智能选股（主用）**:
共用工具函数 `buildKeyWordNew()` 将四层漏斗参数转换为东财 AI 选股 API 的自然语言 keyWordNew 字符串（Dashboard 和 Screener 页面共用同一逻辑，避免 drift）。AI API 返回 `responseConditionList` 展示已解析的条件标签。前端回退判定：若 AI 返回 total>200 且 conditions<3，说明解析不完整，自动回退到结构化 API。

**结构化选股（回退）**:
`buildStructuredFilter()` 将四层漏斗参数转换为东方财富 xuangu 结构化 API 的 filter 字符串，作为 AI 选股不可用时的回退方案。`buildFilterDescription()` 生成可读中文描述，展示在选股结果上方。

**登录态管理**:
AI 选股 API 在无登录态时只能解析 2-3 个条件。通过 `server/.env` 配置 `EASTMONEY_EMAUTH` cookie 值，后端自动注入 `Cookie: emauth=xxx` 请求头。前端通过 `GET /api/stock/xuangu/ai/status` 检测 cookie 配置状态，未配置时显示黄色提示条。

**filter 字符串示例（偏多/回调买入/宽模式）**:
> (TRADE_MARKET_CODE in ("上交所主板","深交所主板","深交所创业板","上交所科创板"))(@LISTING_DATE="OVER1Y")(GOODWILL_TO_NETASSET<30)(PLEDGE_RATIO<60)(ROE_WEIGHT>=8)(DEBT_ASSET_RATIO<=65)(PE9>3)(PE9<=60)(TOTAL_MARKET_CAP>=3000000000)(LONG_AVG_ARRAY="1")

**keyWordNew 示例（偏多/回调买入/宽模式）**:
> 非ST，非停牌，非北交所，非退市，上市时间超过1年，商誉占净资产比例小于30%，质押比例小于60%，营收增速大于等于5%，利润增速大于等于5%，负债率小于等于65%，PE 3~50，市值大于等于30亿，回调买入：均线多头排列 + 量能收缩

**候选股卡片**展示：股票名称、涨跌幅、PE、PB、换手率、主力净流入、量比。点击跳转股票池页面。

**后端接口**: `GET /api/stock/xuangu/structured`（主用，结构化选股 API）

#### 3.1.4 做多窗口速判

3 项自动检测条件，满足即提示多头窗口开启：
1. 指数收盘价 > MA60
2. MA60 连续 3 天拐头向上
3. 涨跌比 ≥ 1.8

#### 3.1.5 宏观因子卡片

Dashboard 显示 5 个宏观因子卡片（PMI / M1-M2 剪刀差 / CPI / GDP / 社融），每个卡片展示最新值、日期和评分信号。下方详情行展示各因子的利好/利空/中性描述。宏观因子数据 6 小时缓存（服务端），前端 60 秒刷新。

#### 3.1.6 行业资金流向 & RS 轮动

Dashboard 统一面板，始终显示资金流向数据，RS 轮动数据在有足够历史时自动展示：

**资金流向（始终显示）**:
- 左列：涨幅 TOP5（三级行业分类），展示行业名称、涨跌幅、主力净流入、领涨股
- 右列：主力净流入 TOP5，展示行业名称、净流入金额、涨跌幅
- 数据源：东方财富 ZJLX API（三级行业分类，`fs=m:90+t:2`），实时刷新

**RS 轮动信号（需 ≥ 5 天历史快照）**:
- 基于板块历史快照（持久化到 `rs_history.json`），计算多周期相对强度排名
- **RS Score** = excess5d × 0.5 + excess10d × 0.3 + excess20d × 0.2（超额收益加权）
- 板块与基准使用等长窗口计算超额收益，避免窗口不对称
- **轮动信号**: TOP10 板块中加速 ≥ 6 → `strengthening`，减速 ≥ 6 → `weakening`，其他 → `mixed`
- **显示**: RS 强势 TOP5 / RS 弱势 TOP5，每行展示板块名称、5日超额收益、RS比率、趋势箭头、当日涨跌幅

策略选股自动注入 TOP3 RS 强势行业作为行业过滤条件。

#### 3.1.7 交易前检查清单

9 项必检项（来源 `PRE_TRADE_CHECKLIST`），以复选框形式呈现：
1. 大盘状态是否适合交易？（对照九维判定 + 决策树）
2. 该股是否通过排雷清单？（对照3.2）
3. 是否有明确买入信号？（对照4.1/4.2/4.3）
4. 止损位是否已设定？（ATR计算 + 条件单）
5. 盈亏比是否 ≥ 2:1？
6. 仓位是否在风险预算内？（5.3公式计算）
7. 是否已设条件单/提醒？（不依赖盘中判断）
8. 是否在冷静状态下决策？（不在开盘/收盘前10分钟冲动）

### 3.4 仓位计算 (Position)

**路由**: `/position`
**用途**: 基于 ATR 的动态仓位计算、持仓管理、行业集中度监控

#### 3.4.1 ATR 动态仓位计算器

**输入**:
- 股票代码（带「获取 ATR」按钮，自动调用 K 线接口计算 ATR(14)）
- 买入价
- 目标价
- 策略选择（趋势突破 / 回调买入 / 底部右侧确认）

**自动计算**:

止损价 = 买入价 − N × ATR (N 由策略决定)

| 策略 | ATR 乘数 N | 回撤比例 | 建议持有期 |
|------|-----------|---------|-----------|
| 趋势突破 | 1.5 | 8% | 1-4 周 |
| 回调买入 | 2.0 | 15% | 2-8 周 |
| 底部右侧确认 | 3.0 | 20% | 1-6 月 |

仓位 = min(风险预算 / 止损比例, 总资金 × 25%)

其中:
- 风险预算 = 总资金 × 2%（单笔最大亏损）
- 25% 为单股仓位上限

盈亏比 = (目标价 − 买入价) / (买入价 − 止损价)

**展示**:
- 止损价、建议仓位（万元）、仓位占比
- 盈亏比数值（建议 ≥ 2:1）
- 止损/止盈可视化条形图

#### 3.4.2 行业集中度

- 按行业分组统计持仓占比
- 某行业 > 30% 时发出警告
- 可视化柱状图

#### 3.4.3 持仓管理表

| 字段 | 说明 |
|------|------|
| 代码 | 股票代码 |
| 行业 | 所属行业 |
| 买价 | 买入价格 |
| 止损价 | 初始止损 |
| 跟踪止盈 | 动态止损价（只能上移，不能下调） |
| 仓位 | 持仓金额 |
| 策略 | 策略名称 |
| 日期 | 开仓日期 |

**操作**:
- 添加持仓：从计算器结果直接添加
- 更新跟踪止盈：仅允许上调
- 删除持仓
- 总资金可配置（默认 100 万），持久化到 localStorage

#### 3.4.4 策略参数参考表

展示三种策略的完整参数对照：ATR 乘数、回撤比例、建议持有期、时间止损规则。

| 维度 | 满分 | 子项 | 权重（有基本面） | 权重（无基本面） |
|------|------|------|-----------------|-----------------|

**行业感知阈值**: PE/PB 覆盖 20 个行业分类（银行/保险/地产/钢铁/煤炭/食品/医药/科技/半导体/军工/新能源/家电/汽车/公用事业/化工/建筑/券商/采掘/农业/交运），每个行业有独立的低估/合理/偏高阈值。负债率覆盖 7 大行业组（银行/保险/地产/券商/公用事业/建筑/通用），使用行业适配阈值判定。

#### 3.6.3 技术面 Tab

**K 线图**: 支持 5 种周期切换（日K/周K/月K/60分/30分），默认日K，250 根 K 线
**技术指标**: 自动计算 MA(5/10/20/60)、MACD(12,26,9)、KDJ(9)、RSI(6/12/24)、BOLL(20,2)
**信号检测**: 6 类信号生成器

| 信号源 | 检测内容 |
|--------|---------|
| MA | 多头/空头排列，MA5/10、MA10/20、MA20/60 金叉/死叉（5/7/10 日回溯窗口） |
| MACD | 真实金叉/死叉（柱状图穿越零轴），零轴位置，顶/底背离，弱信号降权×0.7 |
| KDJ | 超买/超卖区（J<0 与 K/D 超卖互斥），低位金叉(K<30)/高位死叉(K>70)，过滤不可靠交叉（高位金叉/低位死叉），顶/底背离 |
| RSI | 超买(>70/80)/超卖(<20/30)，RSI(6/24) 交叉（5 日回溯窗口），多周期共振（金叉/死叉期间抑制），顶/底背离，超买上下文感知（中长周期未确认时降级为中性） |
| BOLL | 趋势感知：上升趋势沿上轨=偏强，下降趋势沿下轨=偏弱；突破/跌破布林带；震荡市触及上轨=超买预警；带宽收口变盘预警 |
| 量价 | MA20 趋势感知：上升趋势缩量回调=洗盘(bullish)，下降趋势缩量下跌=无量阴跌(bearish)；连续缩量上涨=量价背离(bearish)；单日放量/缩量异常，连续 3 日量能趋势 |

背离检测使用峰值/谷值识别算法（3 日回看窗口），用于 MACD、KDJ、RSI 三个维度。

#### 3.6.4 基本面 Tab

**估值区间**: PE/PB 历史分位可视化（渐变条 + 当前位置标记），分位计算排除当期数据

**财务趋势图**: ECharts 双轴图（柱状营收/净利润 + 折线毛利率/ROE），展示近 8 个季度数据

**核心指标卡片**（3 组）:

| 组别 | 指标 |
|------|------|
| 估值 | PE(TTM)、PB(MRQ) — 行业感知阈值着色（20 个行业独立阈值） |
| 盈利能力 | ROE、毛利率、净利率 — 分级着色 |
| 成长与安全 | 营收增长、净利增长、负债率（行业适配阈值，7 组）、经营现金流（每股）、现金流/净利比、总市值 — 分级着色 |

**数据来源**:
- PE/PB/总市值: 东方财富 `RPT_VALUEANALYSIS_DET` 报表
- 财务指标: 东方财富 `ZYZBAjaxNew` 接口（ROE/EPS/营收/净利/毛利率/负债率等）
- 历史 PE: 周K线 + EPS 推算，TTM EPS 采用累计 EPS + 上年同期桥接算法

**后端接口**: `GET /api/stock-analysis/fundamental?code=xxx`

#### 3.6.5 资金面 Tab

**量价分析**: 量能信号标签（如"温和下跌"）与标题同行显示。服务端结合 20 日趋势方向判断：上升趋势缩量下跌→"缩量回调（洗盘）"，下降趋势缩量下跌→"缩量下跌（弱势）"，上升趋势缩量微涨→"缩量整理（蓄势）"，下降趋势缩量微涨→"缩量调整（弱势）"。

**主力资金**:
- 4 项指标卡片：今日净流入、净流入比率、主力/散户流向、近10日趋势
- 综合评分：`combinedPct = todayPct × 0.4 + avg5Pct × 0.6`（今日主力净占比 40% + 近 5 日均占比 60%）
- 评分梯度：>5→8分, >2→6分, >0.5→4分, ±0.5→3分(中性), ≥-2→2分, ≥-5→1分, <-5→0分
- 主力10日趋势加成：近5日均净流入 > 前5日 → 评分+1，反之 -1

**北向资金**:
- 4 项指标卡片：持股数量、持仓市值、占流通股比、持股变动
- ECharts 双轴图：持仓市值（柱状）+ 占流通股比（折线）
- 数据来源: 东方财富 `RPT_MUTUAL_HOLDSTOCKNDATE_STA_NEW` 报表

**融资融券**:
- 4 项指标卡片：融资余额、融资净买入、融券余额、余额变化率
- ECharts 双轴图：融资余额（柱状）+ 融资净买入额（折线，正值红色负值绿色）
- 做空信号检测：融券余额日环比增 >20% → 评分 -2，>10% → 评分 -1（基值 >100 万元过滤噪音）
- 多维评分：融资余额增长率(60%, 满分5) + 融资净买入比率(40%, 满分5)，合计满分 5
- 余额阈值：≥2%→大幅上升, ≥0.5%→上升, >0.3%→微升, ±0.3%→持平, ≥-0.5%→微降, >-2%→下降, ≤-2%→大幅下降
- 数据来源: 东方财富 `RPTA_WEB_RZRQ_GGMX` 报表，30 天明细

**股东户数（筹码趋势）**:
- 4 项指标卡片：最新股东户数、环比变化、变化趋势、筹码集中度评价
- ECharts 折线图：股东户数变化趋势
- 筹码趋势评分(0-5，9 档)：≤-10%→5, ≤-5%→4, ≤-3%→4, ≤-1%→3, ±1%→3(稳定), ≤3%→2, ≤5%→2, ≤10%→1, >10%→0
- 多周期趋势修正：连续3期间变化幅度均 >1% 且方向一致时额外 ±1 加成/扣分
- 数据来源: 东方财富 `RPT_HOLDERNUM_DET` 报表

**后端接口**:

## 4. 后端接口规范

## 7. 交易规则体系（核心业务逻辑）

### 7.1 九维大盘判定算法（v7.3）

来源: `src/utils/marketJudge.js`（前 7 维度前端计算）+ `server/analysis.js`（第 8 维度服务端计算）

**维度与权重**:
| # | 维度 | 权重 | 强信号阈值 | 数据来源 |
|---|------|------|-----------|---------|
| 1 | MACD | 1.0 | 背离 → 2.0 × confidence(0.1~1.0) | 三指数(上证/深证/创业板) MACD(12,26,9)，多指数投票（同向×1.2/分化×0.7），多尺度背离检测+幅度过滤+时间衰减 |
| 2 | 涨跌家数 | 1.5 | 比值 ≥ 2 或 ≤ 0.5 → 2.0 | 沪深京A股实时涨跌统计，牛熊分界线 1.8/0.55，过渡档 1.5/0.67 降权×0.7 |
| 3 | RSI | 1.0 | 背离 → 2.0 × confidence(0.1~1.0) | 三指数 RSI(14)，多指数投票，RSI≥60 偏强（×0.7），<40 偏弱（×0.7） |
| 4 | 融资余额 | 1.2 | 回归斜率 > ±0.5%/日 → 2.0 | 近 10 日融资余额线性回归 |
| 5 | 量价配合 | 1.3 | OBV 背离 → 2.0 × confidence(0.1~1.0) | 三指数 OBV 趋势，多指数投票，归一化斜率 ±0.05% 死区 |
| 6 | 北向资金 | 1.5 | 活跃度比 ≥ 1.4 或 ≤ 0.6 → 2.0 | 近 20 日北向成交额 |
| 7 | 涨跌停 | 1.3 | 并行评分 net ≥ 4 或 ≤ -4 → 2.0 | 当日涨跌停+连板+板块分布+自然涨停率，跌停0家时按量级映射，含集中度和自然率惩罚系数 |
| 8 | 宏观因子 | \|score\| × 0.5 | 服务端 5 因子评分 | PMI/M1-M2/CPI/GDP/社融 |

输入: 指数数据（K 线、MACD、RSI）、涨跌家数（含历史）、融资余额、量价关系、北向资金、涨跌停统计、前一市场状态

输出:
```typescript
{
  status: 'bull' | 'bull-lean' | 'neutral' | 'bear-lean' | 'bear'
  label: string       // 如「牛市」「偏多」
  tag: string         // CSS class
  maxPosition: string // 如 '80-100%'
  strategy: string    // 推荐策略
  strategyName: string // 推荐策略名称
  signals: object[]   // 各维度判断结果（含 weight, bull, bear, desc, divergence, subSignals）
  score: { bull, bear, neutral, bullW, bearW } // 信号计数 + 加权总分
  confirmed: boolean  // bullW 或 bearW ≥ 3.5
  hysteresisState: {  // 迟滞状态（v7.2）
    status: string    // 当前状态
    crossCount: number // 跨方向连续确认计数
    lastFlipDate: string | null // 上次跨方向翻转日期
  }
  longWindow: {       // 做多窗口速判
    conditions: object[]
    allPass: boolean
    message: string
  }
}
```

### 7.2 仓位计算公式

来源: `src/utils/position.js`

```
止损距离 = N × ATR(14)
止损价 = 买入价 − 止损距离
止损比例 = 止损距离 / 买入价
风险预算 = 总资金 × 2%
单笔仓位 = min(风险预算 / 止损比例, 总资金 × 25%)
盈亏比 = (目标价 − 买入价) / 止损距离
跟踪止盈初始价 = 买入价 × (1 − 回撤比例)
```

### 7.3 ATR 计算方法

```
True Range = max(当日最高 − 当日最低, |当日最高 − 昨收|, |当日最低 − 昨收|)
ATR(14) = SMA(TR, 14)  // 取最近 15 根 K 线，计算后 14 个 TR 的均值
```

## 8. 非功能性需求

### 8.1 性能

- 股票池行情每 `REFRESH_INTERVAL`（默认 10 秒）自动刷新，批量查询
- 服务端 API 缓存：涨跌家数 30 秒、其余接口 5 分钟，减少对东方财富的请求频率
- Dashboard 涨跌家数独立 30 秒刷新定时器，确保盘中数据实时性
- 市场状态带惯性（hysteresis）机制，避免相邻判定周期之间状态频繁跳变
- 东方财富 API 请求串行执行，避免并发触发限流
- 请求超时 8 秒自动中断，不会无限等待
- Sparkline/KlineChart 使用 Canvas 渲染，避免 DOM 开销
- K 线数据客户端缓存（klineCache），避免重复请求
- 搜索输入 300ms 防抖

### 8.2 容错

- API 请求失败时优先使用缓存数据，不进行降级处理
- 涨跌家数采用东方财富 ulist API（push2.eastmoney.com/api/qt/ulist/get），单请求汇总沪A(1.000002)+深A(0.399002)+北交所(0.899050) 的涨(f104)/跌(f105)/平(f106)家数，30秒缓存
- 东方财富 push2 系列域名（push2/push2his）不稳定时，自动切换到腾讯/新浪备用数据源（`server/fallback.js`）
- 北向资金使用 datacenter-web 的 RPT_MUTUAL_DEALAMT 报表（主用），失败时自动回退到证券时报 stcn 数据源（`info.stcn.com/dc/sjb/newindex.jsp?p=xcxBszjcjetusj`）
- 选股功能三级回退：AI 自然语言选股（`np-tjxg-g.eastmoney.com`，主用）→ 结构化选股 API（`data.eastmoney.com`，回退）→ 本地两步筛选（`/api/stock/screen`，兜底）
- AI 选股 API 在无登录态（`EASTMONEY_EMAUTH` cookie）时只能解析 2-3 个条件；前端回退判定：AI 返回 total>200 且 conditions<3 时自动切换到结构化 API
- AI 选股结果以 keyWordNew 内容为缓存 key，TTL 5 分钟
- 单个数据请求失败不影响其他数据的正常展示
- 图表加载中/加载失败/无数据三种状态明确区分
- 北向资金、融资余额、涨跌停数据持久化到 localStorage 作为本地缓存
- 前一市场状态持久化到 localStorage，页面刷新后惯性机制可继续工作

### 8.3 兼容性

- 现代浏览器（Chrome / Edge / Firefox / Safari）
- 移动端响应式适配（768px 断点）
- Retina 屏幕 DPR 适配（Canvas）

### 8.4 数据安全

- 所有用户数据（自选、持仓、交易记录）仅存储在浏览器 localStorage
- 无服务端数据库，无用户注册/登录
- API 仅为东方财富公开数据的代理转发

### 8.5 可用性

- 无网络时：已缓存的自选列表和交易记录仍可查看（离线只读）
- API 失败时：静默降级，不阻塞页面操作
- 删除操作需二次确认
- 页面切换时自动启停对应定时器，不浪费资源
